Number Systems

Number Systems

Integers could be represented in any base.

In a base system, digits are limited to 0 to (base - 1).

Decimal and Binary Number Systems

Base 10 is natural for humans.

Base 2 is what computers use.

Base Conversions

Decimal to Binary

Divide the decimal number by 2. Store the remainder.

Continuously divide the quotient by 2 until 0.

Read the remainders in reverse acquisition order.

Decimal to Binary

Convert 125 to binary.

						
	125 / 2 = 62 r. 1
	 62 / 2 = 31 r. 0
	 31 / 2 = 15 r. 1
	 15 / 2 = 7  r. 1
	  7 / 2 = 3  r. 1
	  3 / 2 = 1  r. 1
	  1 / 2 = 0  r. 1

Answer: 111 1101

Binary to Decimal

Going from right to left, multiply each bit with increasing powers of two, starting with \(2^0\).

Add the products.

Binary to Decimal

Convert \(111 1101_2\) to decimal.

						
	1 * 1  = 1
	0 * 2  = 0
	1 * 4  = 4
	1 * 8  = 8
	1 * 16 = 16
	1 * 32 = 32
	1 * 64 = 64

	1 + 0 + 4 + 8 + 16 + 32 + 64 = 125

Hexadecimal

Base 16 number system

Uses the numbers 0-9 with letters A-F for 10-15 as digits.

Decimal to Hexadecimal

Divide the decimal number by 16. Store the remainder.

Continuously divide the quotient by 16 until 0.

Read the remainders (and replace with corresponding hex digit) in reverse acquisition order.

Why use hexadecimal?

It is a shorthand for the binary number system!

Representing Negative Integers

Representing Negative Integers

  • Sign-magnitude
  • One's Complement
  • Two's Complement

Sign-Magnitude

Allocate one bit to represent sign (0 for positive, 1 for negative). Use the remaining bits to represent the magnitude.

Sign-Magnitude

In an 8-bit computer, below are the sign-magnitude representations for some signed integers.

                        
    -125 : 1111 1101
     125 : 0111 1101
      36 : 0010 0100
     -36 : 1010 0100

One's Complement

The ones' complement of a binary number is obtained by inverting all the bits in the binary representation of the number.

One's Complement

In an 8-bit computer, below are the one's complement representations for some signed integers.

                        
    -125 : 1000 0010
     125 : 0111 1101
      36 : 0010 0100
     -36 : 1101 1011

Representing Zero

Both sign-magnitude and one's complement have two representations for zero. This poses some ambiguities in binary operations.

Representing Zero

                        
    0 (s/m) : 1000 0000 (-0)
              0000 0000 (+0)

    0 (1c)  : 0000 0000 (+0)
              1111 1111 (-0)

Two's Complement

Removes the double representation of zero in one's complement.

Two's Complement

Apply One's Complement, then add 1.
